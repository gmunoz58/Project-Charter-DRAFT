Project Tool Selection Explanation

Asana I decided to go with Asana because it‚Äôs the tool I have the most experience with since we use it at my job. Additionally, I believe it best supports my hybrid iterative methodology since it allows me to set up clear deliverables with timelines while still supporting iterative improvements through subtasks, comments, and feedback loops.
Key features:
  • Timeline view has a Gantt like display for deliverables and dependencies across 12 weeks.
  • Sections and phases can separate Plan, Execute, and Deliver phases.
  • Subtasks and comments are perfect for iteration cycles and mentor/peer feedback.
  • Milestones and deadlines keep me accountable at key points in my project.
Asana has a free tier that supports task assignments, subtasks, boards, and basic timelines. There is a premium version for more reporting features, but the free version is plentiful enough for a solo project like mine.
